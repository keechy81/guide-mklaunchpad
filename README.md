# guide-mklaunchpad

一个用于归档和发布独立 HTML 页面的通用仓库。

## 项目简介

本仓库主要用于存放和发布多个独立的 HTML 页面，不针对任何单个域名或具体网站。每个页面都是自包含的，可直接在浏览器中打开运行，适合作为页面原型、功能演示、工具界面或静态内容的存档。

## 目录结构

```
guide-mklaunchpad/
├── pages/            # 存放独立 HTML 页面的目录
│   ├── page-1.html
│   └── page-2.html
├── assets/           # 页面依赖的静态资源（可选）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md         # 本文件
```

- `pages/`：核心目录，所有独立 HTML 页面均放置于此。
- `assets/`：用于存放页面共用的 CSS、JavaScript、图片等资源，也可根据需要在页面内内联。

## 页面归档说明

- 每个 HTML 页面应尽量保持独立，减少对外部资源的依赖。
- 页面命名建议使用有意义的英文或拼音，便于识别。
- 如需引用 `assets/` 中的资源，请使用相对路径。
- 页面内不应包含指向具体商业网站、服务或营销内容的链接。

## 维护说明

- 添加新页面时，直接放入 `pages/` 目录，并在本 README 中更新页面列表（可选）。
- 修改已有页面时，请保持原有文件名不变，以便链接稳定。
- 删除页面时，建议在 README 中移除对应记录，避免出现死链接。
- 如页面涉及敏感或过时内容，请及时移除或替换。

## 贡献

欢迎提交 Pull Request 添加新的独立 HTML 页面。请确保页面内容合规、不包含恶意代码，并尽量遵循上述归档规范。
